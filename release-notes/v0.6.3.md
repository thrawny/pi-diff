# pi-diff 0.6.3

## Summary

Fixes extension load failure from published/local installs that pulled a nested Pi runtime into the extension install.

## What changed

- Removed Pi host packages from published `peerDependencies` so npm does not auto-install nested copies.
- Added package `main`/`exports` pointing at `./dist/index.js`.
- Changed review-diff helpers to avoid top-level runtime imports from `@earendil-works/pi-coding-agent` and `@earendil-works/pi-tui`.
- Lazy-load optional TUI review pane code only when `/review-diff --tui` needs it.
- Use local clipboard command fallbacks instead of importing `copyToClipboard` from the Pi host package.

## Why

The crash looked like:

```text
Cannot find module .../@earendil-works/pi-ai/dist/index.js/base
Require stack:
- .../@earendil-works/pi-agent-core/dist/agent.js
```

That happened because npm installed a nested `@earendil-works/pi-coding-agent` for the extension. Loading Pi host packages from inside an extension is unsafe and can break module resolution across Pi versions.
